3.Constructive Logic

Exclusive middle law:

$$
\vdash P\lor \neg P
$$

It’s strange that no need to inspect the internal structure of P. Consider:

Proof. Consider $p=q=\sqrt2$, if $\sqrt2 ^ \sqrt 2$ is rational, proof completes.

Else $\sqrt2 ^ \sqrt 2$ is irrational, then $(\sqrt2 ^ \sqrt 2) ^ \sqrt2= 2$.

Qed.

NNF

  • 没有蕴含

  • 所有的否定只能出现在原子命题前

Syntax:
$$
\begin{align*}
P :: &= \ p \
&\ \mid \neg p \
&\ \mid T \
&\ \mid \bot \
&\ \mid P \lor P \
&\ \mid P \land P
\end{align*}
$$
例如：$P1 \land P2 \lor \neg P3$

注意到有
$$
P \rightarrow Q = \neg P \lor Q
$$
转换为NNF的规则为
$$
\begin{array}{ll}
C(p) &= p \
C(\neg p) &= \neg p \
C(\neg\neg P) &= C(P) \
C(P \land Q) &= C(P) \land C(Q) \
C(P \lor Q) &= C(P) \lor C(Q) \
C(\neg(P \land Q)) &= C(\neg P) \lor C(\neg Q) \
C(\neg(P \lor Q)) &= C(\neg P) \land C(\neg Q)
\end{array}
$$

CNF

  • 内层为析取，外层为合取

Syntax:
$$
\begin{align*}
P :: &= D \land P\
D :: &= D \lor A\
A :: &= p \
& \ \mid \neg p \
&\ \mid T \
&\ \mid \bot \
\end{align*}
$$
例如：$(P1 \lor P2) \land (\neg P3 \lor P4)$

转换为CNF的规则为：
$$
\begin{array}{ll}
C(p) &= p \
C(\neg p) &= \neg p \
C(P \land Q) &= C(P) \land C(Q) \
C(P \lor Q) &= D(C(P) , C(Q)) \

D(P=P1 \land P2, Q) &= D(P1, Q) \land D(P2, Q) \
D(P=P, Q1 \land Q2) &= D(P, Q1) \land D(P, Q2) \
D(P, Q) &= P \lor Q \

\end{array}
$$
Remarks:

  • Think both C( ) and D( ) as compilers;

  • The D( ) function basically pushes the / connective deeper into a given proposition P.

DPLL算法流程

  • Splitting rule
  • Unit propagation

dpll(P){
  if(P==T)
    return sat;
  if(P==F)
    return unsat;

  unitProp(P);  // unit prop and simplify P
  x = select_atomic(P); // choose an atomic prop
  if(dpll(P[x|->T])) // splitting
    return sat;
  return dpll(P[x|->F]);
}

Fourier-Motzkin

  • Normal form:

Normalize to (only ≥0 ), look for some variable $x_i$, which is of positive and negative occurrences :
$$
\begin{cases}

x_i + P_1(x) \geq 0 \
…\
x_i + P_m(x) \geq0 \

-x_i + Q_n(x) \geq 0 \

\end{cases}
$$
Where $P_j (x), Q_k (x)$ and $R(x)$ don’t contain $x_i$. Eliminating $x_i$, and thus simplifying to:
$$
\begin{cases}

x_i + P_1(x) \geq 0 \
…\
x_i + P_m(x) \geq0 \
-x_i + Q_1(x) \geq0 \
…\
-x_i + Q_n(x) \geq 0 \
R(x) \geq 0

\end{cases}

\to

\begin{cases}
P_1(x)+Q_1(x)\geq0\…\ P_1(x)+Q_n(x)\geq0\ P_2(x)+Q_1(x)\geq0\…\ P_2(x)+Q_n(x)\geq0\

P_m(x)+Q_1(x)\geq0\

\end{cases}
$$

  • Normal form:

$$

It is common to represent the coefficients in the input problem using an $m-by-(n + m)$ matrix $A$. The variables $x_1, . . . , x_n, s_1, . . . , s_m$ are written as a vector $x$. Following this notation, our problem is equivalent to the existence of a vector $x$ such that
$$
\begin{equation*}
A \mathbf{x}=0 \quad \text { and } \quad \bigwedge_{i=1}^{m} l_{i} \leq s_{i} \leq u_{i}
\end{equation*}
$$

给还没确定值的自变量定值为 0 (初始自变量)或边界值(换轴后自变量的 bound)，看看是否满足 Tableau 约束，若不满足条件则尝试换轴(pivoting)；pivoting 后要满足 Tableau 的约束从而更新 Tableau，然后重复本步骤

  • Algorithm：GENERAL-SIMPLEX

Algorithm：GENERAL-SIMPLEX
Input：A linear system of constraints $S$
Output：”Satisfiable” if the system is satisfiable,and “Unsatisfiable” otherwise

  1. Transform the system into the general form
    $$
    \begin{equation*}
    A \mathbf{x}=0 \quad \text { and } \quad \bigwedge_{i=1}^{m} l_{i} \leq s_{i} \leq u_{i}
    \end{equation*}
    $$

  2. Set $\mathcal{B}$ to be the set of additional variables $s_1,…,s_m$.

  3. Construct the tableau for $A$.

  4. Determine a fixed order on the variables.

  5. If there is no basic variable that violates its bounds,return “Satisfiable”. Otherwise, let $x_i$ be the first basic variable in the order that violates itsbounds.

  6. Search for the first suitable nonbasic variable , in the order for pivoting with $x_i$;.If there is no such variable，return “Unsatisfiable”.

  7. Perform the pivot operation on $x_i$; and $x_j$.

  8. Go to step 5.

  • code
simplex(){
  tab = constructTableau();
  for(each additional var si){
    if(si violates its constraint){
      if(there is a suitable xj)
        pivot(si, xj);
      else return UNSAT;
    }
  }
  return SAT;
}

Branch & Bound(ILP)

Solve the problem for $x, y \in R$

  • No solution, then UNSAT
  • Find a solution $[x=r_0, y=r_1]$
    • if $r_0,r_1 \in \mathbb{Z}$,SAT!
    • Else suppose $c \in \mathbb{R}$, then we add two branches:
      • $S \cup [x ≥ \lceil c \rceil]$
      • $S \cup [x ≤ \lfloor c \rfloor]$
branchBound(S){
  res = simplex(S);
  if(res==UNSAT) // prune
    return UNSAT;
  if(res are integers)
    exit(SAT);   // deep return
  c = select(x of real value);
  return branchBound(S ∪[x ≥ ⌈𝑐0⌉])|| // backtrack
                 branchBound(S ∪[x ≤ ⌊𝑐⌋]);
}

Syntax

$$
\begin{align*}
\mathbb{\oplus}::&= + \mid - \mid ^* \mid / \mid << \mid >> \mid >>> \mid & \mid | \mid xor\
E :: &= x \mid c \mid E \oplus E \mid \neg E \
R :: &= E=E \mid E <= E \mid E < E\

\end{align*}
$$

x, y = BitVecs(‘x y’, 8) 
solve(x+y == 1024) 
# why Z3 output:[x=0, y=0] ?

Motivation

Bit vectors are represented by the corresponding propositions

  • Is a=1 ∧ b=2 ∧ a&b=1 satisfiable?

$$
\begin{align*}
𝑎=1&⟺𝑎_0∧¬ 𝑎_1∧… \
𝑏=2&⟺¬𝑏_0∧𝑏_1∧… \
𝑎&𝑏=1&⟺(𝑎_0∧𝑏_0)∧…
\end{align*}
$$

Bit vector operations are compiled to the semantically equivalent operations on propositions

  • Also called constraints

These constraints are handled by a SAT

This is the bit-blasting algorithm

C = {}; // a set of all generated constraints

// two main passes:
// 1. blast each proposition;
// 2. generate constrains
bitBlast(P){
  // convert the proposition to atomic bools
  blastProp(P);
  // generate constraints
  genConsProp(P);
}

1.blasting：其实就是给每个可计算出值的节点（即除了 ∧,=∧,= 以外的节点）赋予 n 长度的 boolean 数组（bit-vector），以便后面产生约束用

// blastProp() will blast each proposition P
blastProp(P){
  if P is (e1=e2){
    // for atomic propositions, crawl through
    // expressions
    blastExp(e1);
    blastExp(e2);
  }else if P is (P1 ∧ P2){ // trivial recursion
    blastProp(P1); 
    blastProp(P2);
  }
}
// genCons() will generate constraints
blastExp(e){
  if e is x:
    // a vector of boolean variables
    return (b0, b1, …, bn); 
  if e is c:
    return (b0, b1, …, bn);
  if e is e1+e2:
    (b0, …, bn) = blastExp(e1);
    (c0, …, cn) = blastExp(e2);
    return (d0, …, dn);  // attach to e1+e2
  // other cases are similar
}

2.generating constraints：产生这些 bit-vector 互相之间的约束条件，以便之后扔进 DPLL 计算 SAT

// genConsProp() will generate constraints
genConsProp(P){
  if P is (e1=e2):
    genConsExp(e1);
    genConsExp(e2);
    C ∪= {x0=y0, …, xn=yn};
  else if P is (P1 ∧ P2):
    genConsProp(P1);
    genConsProp(P2);
}
// genExp() will generate constraints for exp
genConsExp(e){
  switch(e){
  case (x&y): 
    C ∪= {z0=x0 ∧ y0, z1=x1 ∧ y1, …, zn=xn ∧ yn}
    break;
  case (x|y):
    C ∪= {z0=x0 ∨ y0, z1=x1 ∨ y1, …, zn=xn ∨ zn}
    break;
  case (~x):
    C ∪= {z0=~x0, z1=~x1, …, zn=~xn}; break;
  case (x+y): 
    c0=F ∧
      r0=xor(x0,y0,c0) ∧ c1=(x0 ∧ y0)\/(xor(x0,y0) ∧ c0)
      r1=xor(x1,y1,c1) ∧ c2=(x1 ∧ y1)\/(xor(x1,y1) ∧ c1)
      …
  case (x-y):genConsExp(x+(-y))
  case (x*y):genConsExp(y+y+...+y)
  case (x/y):genConsProp(y!=0∧d*y+r=x∧r<y)
  }
}

$$

Semantics

The rules to eliminate a pointer T are:
$$
\begin{align*}

⟦x⟧ &= H(S(x)) \
⟦T + E⟧ &= ⟦T⟧ + ⟦E⟧ \
⟦&x⟧ &= S(x) \
⟦&T⟧ &= ⟦T⟧ \
⟦T⟧ &= H(⟦T⟧) \
⟦NULL⟧ &= 0 \
\end{align*}
$$

sat(P){
  P' = ⟦P⟧;	// Pointer semantics
  return sat(P');

Pure variables

We call a variable x pure

  • if x’s address is not taken of: &x
  • else the variable x is called escaped

We can introduce a new sort a memory : $R(x)$

  • The standard translation: $H(S(x))$
  • For pure vars, new translation: $R(x)$

example:
$$
Prove :x=y → y=x \

⟦𝑥=𝑦 → 𝑦=𝑥⟧ \
⟦𝑥=𝑦⟧→⟦𝑦=𝑥⟧ \
𝐻(𝑆(𝑥))=𝐻(𝑆(𝑦))→𝐻(𝑆(𝑦))=𝐻(𝑆(𝑥))
$$
引入pure variables
$$
Prove :x=y → y=x \

⟦𝑥=𝑦 → 𝑦=𝑥⟧ \
⟦𝑥=𝑦⟧→⟦𝑦=𝑥⟧ \
𝑅(𝑥)=𝑅(𝑦)→𝑅(𝑦)=𝑅(𝑥) \
$$
